01

HTML5 문서의 기본 구조

모든 HTML 파일은 아래의 뼈대(skeleton)로 시작합니다. VS Code에서 !를 입력하고 Tab을 누르면 자동으로 생성됩니다.

skeleton.html 
<!DOCTYPE html>            <!-- "이 문서는 HTML5입니다" 선언 -->
<html lang="ko">            <!-- 문서의 시작, 언어 설정 -->
<head>                       <!-- 브라우저에게 전달하는 정보 영역 -->
    <meta charset="UTF-8">  <!-- 한글 깨짐 방지 (필수!) -->
    <title>페이지 제목</title> <!-- 브라우저 탭에 표시되는 이름 -->
</head>
<body>                       <!-- 사용자에게 보이는 내용 영역 -->
    여기에 내용을 작성합니다
</body>
</html>

핵심 개념

<!DOCTYPE html> → 이 문서가 HTML5라는 것을 브라우저에 알려주는 선언문입니다.

<head> → 사용자 눈에 보이지 않는 메타 정보(문자 인코딩, 제목, CSS 연결 등)를 넣는 곳입니다.

<body> → 사용자 눈에 실제로 보이는 모든 내용이 들어가는 곳입니다.

초보자 주의

태그는 반드시 열고 닫아야 합니다. <h1>...</h1> 처럼요. 단, <br>, <hr>, <img>, <input> 같은 태그는 닫는 태그가 없는 빈 태그(void element)입니다.

02

글자 태그

제목과 본문 글자 태그

h는 heading(제목), p는 paragraph(단락), br은 break(줄 바꿈), hr은 horizontal rule(수평 줄)의 약자입니다.

태그의미특징
<h1> ~ <h6>제목 (1이 가장 큼)블록 요소, 자동 줄바꿈
<p>문단(본문)블록 요소, 문단 사이 간격 자동 생성
<br>줄 바꿈빈 태그 (닫는 태그 없음)
<hr>수평선빈 태그, 내용 구분선으로 사용
text_basic.html 
<body>
    <h1>홍차</h1>
    <hr>
    <h2>정의</h2>
    <p>홍차는 백차, 녹차, 우롱차보다 더 많이 발효된 차의 일종이다.</p>
    <br>
    <h2>등급</h2>
    <p>홍차는 여러 가지로 등급이 매겨진다.</p>
</body>

앵커 태그 (a 태그 = 하이퍼링크)

<a> 태그는 anchor(닻)의 약자로, 다른 페이지나 같은 페이지 내 특정 위치로 이동하는 링크를 만듭니다. href 속성에 이동할 주소를 적어줍니다.

link_example.html 
<!-- 1. 외부 페이지로 이동 -->
<a href="https://www.google.com">Google</a>

<!-- 2. 같은 페이지 내부 이동 (id 활용) -->
<a href="#section1">첫 번째 섹션으로</a>
...
<h2 id="section1">첫 번째 섹션</h2>

<!-- 3. 빈 링크 (아직 이동할 곳이 없을 때) -->
<a href="#">메뉴</a>

핵심 포인트

href="#아이디" 형태로 페이지 내부 이동이 가능합니다. 이때 대상 요소에 id="아이디"가 있어야 합니다.

id 속성은 페이지 내에서 고유해야 합니다 (중복 사용 금지 = 웹 표준 위반).

아직 링크할 곳이 없을 때는 href="#"을 사용하여 웹 표준을 지킵니다.

추가 보충: target 속성

PPT에는 없지만, 실무에서 자주 쓰는 속성입니다.

<a href="..." target="_blank"> → 새 탭에서 열기

<a href="..." target="_self"> → 현재 탭에서 열기 (기본값)

글자 모양 태그

텍스트에 시각적 효과를 주는 태그입니다. 단독으로 쓰거나, 제목/본문 태그 안에 넣을 수 있습니다.

태그효과영어 원뜻
<b>굵은 글자bold
<i>기울어진 글자italic
<small>작은 글자small
<sub>아래 첨자subscript (예: H₂O)
<sup>위 첨자superscript (예: X²)
<ins>밑줄 글자insert
<del>취소선 글자delete

주의: 글자 모양 태그 내부에 h1~h6, p 태그를 넣으면 웹 표준 위반!

올바른 예: <h1><b>제목</b></h1> (h1 안에 b를 넣는 것은 OK)

잘못된 예: <i><h1>제목</h1></i> (i 안에 h1 → 웹 표준 위반!)

추가 보충: 시맨틱한 글자 태그

실무에서는 시각적 태그보다 의미를 가진 태그를 선호합니다.

<strong> = <b>와 동일하게 굵게 표시되지만, "중요한 텍스트"라는 의미를 가짐

<em> = <i>와 동일하게 기울어지지만, "강조된 텍스트"라는 의미를 가짐

검색엔진(SEO)이 <strong>, <em>을 더 잘 인식합니다.

03

목록 태그

웹 사이트의 내비게이션 메뉴는 목록 태그로 만듭니다. Daum, Naver 같은 사이트의 상단 메뉴가 모두 목록 태그 기반입니다.

태그의미결과
<ul>순서 없는 목록 (unordered list)● 점으로 표시
<ol>순서 있는 목록 (ordered list)1. 2. 3. 숫자로 표시
<li>목록 항목 (list item)각 항목 하나

순서 없는 목록 (ul)

<ul>
    <li>사과</li>
    <li>바나나</li>
    <li>오렌지</li>
</ul>

순서 있는 목록 (ol)

<ol>
    <li>사과</li>
    <li>바나나</li>
    <li>오렌지</li>
</ol>

중첩 목록 (목록 안에 목록)

list_nested.html 
<ul>
    <li>
        <b>과일</b>
        <ol>
            <li>사과</li>
            <li>바나나</li>
        </ol>
    </li>
    <li>
        <b>채소</b>
        <ol>
            <li>상추</li>
            <li>양배추</li>
        </ol>
    </li>
</ul>

결과: 바깥은 ● 점 목록, 안쪽은 1. 2. 번호 목록으로 표시됩니다.

04

테이블 태그

태그의미영어
<table>표 전체table
<thead>표의 머리 영역table head
<tbody>표의 몸통 영역table body
<tr>행 (가로줄 하나)table row
<th>제목 셀 (굵고 가운데 정렬)table heading
<td>일반 데이터 셀table data
table_basic.html 
<table border="1">
    <thead>
        <tr>
            <th></th>
            <th>월</th>
            <th>화</th>
            <th>수</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>1교시</td>
            <td>영어</td>
            <td>국어</td>
            <td>과학</td>
        </tr>
    </tbody>
</table>

셀 병합: colspan과 rowspan

colspan = 가로(열)로 셀 합치기, rowspan = 세로(행)로 셀 합치기

table_span.html 
<table border="1">
    <tr>
        <th colspan="2">지역별 홍차</th>   <!-- 2칸 가로 병합 -->
    </tr>
    <tr>
        <th rowspan="3">중국</th>      <!-- 3칸 세로 병합 -->
        <td>정산소종</td>
    </tr>
    <tr><td>기문</td></tr>              <!-- 중국 셀이 차지하므로 td 1개만 -->
    <tr><td>운남</td></tr>
</table>

병합 시 주의

rowspan으로 셀을 세로 병합하면, 병합된 아래 행에서는 그 열에 해당하는 <td>를 적지 않습니다. 처음에 가장 헷갈리는 부분이니 직접 VS Code에서 여러 번 실습해 보세요.

05

미디어 태그

이미지 태그 (img)

<img>는 빈 태그입니다 (닫는 태그 없음). 반드시 src(경로)와 alt(대체 텍스트)를 넣어야 합니다.

속성의미예시
src이미지 파일 경로"photo.jpg" 또는 웹 URL
alt이미지가 안 뜰 때 대체 텍스트"펭귄 사진"
width가로 크기 (px 또는 %)"300"
height세로 크기"200"
<img src="Penguins.jpg" alt="펭귄" width="300">
<img src="Nothing.jpg" alt="그림이 존재하지 않습니다.">

alt 속성은 반드시 작성!

시각장애인을 위한 스크린 리더(Screen Reader)가 alt 텍스트를 읽어줍니다. 웹 접근성을 위해 반드시 넣어야 하며, 빈 문자열 alt=""이라도 넣는 것이 좋습니다.

오디오 & 비디오 태그

HTML5에서 플러그인 없이 멀티미디어를 재생할 수 있습니다.

속성의미
src파일 경로
controls재생 컨트롤러 표시
autoplay자동 재생
loop반복 재생
preload미리 데이터 불러올지 여부
poster (video만)로딩 중 보여줄 이미지
media.html 
<!-- 오디오 -->
<audio controls>
    <source src="music.mp3" type="audio/mp3">
    <source src="music.ogg" type="audio/ogg">
</audio>

<!-- 비디오 -->
<video controls poster="thumbnail.jpg">
    <source src="video.mp4" type="video/mp4">
    <source src="video.webm" type="video/webm">
</video>

왜 source 태그를 여러 개 쓸까?

브라우저마다 지원하는 미디어 포맷이 다릅니다. mp3/mp4는 대부분 지원하지만, ogg/webm도 함께 넣으면 더 넓은 호환성을 확보할 수 있습니다.

06

입력 양식 태그 (Form)

form 태그 기본 구조

<form>으로 영역을 만들고, 내부에 <input> 등의 입력 요소를 배치합니다. 데이터를 입력 후 제출(submit) 버튼을 누르면 지정된 방식으로 서버에 전달됩니다.

속성의미값 예시
method전송 방식"get" 또는 "post"
action데이터 보낼 서버 주소"/submit" 또는 URL

GET 방식

URL 주소에 데이터가 붙어서 전송됩니다. 주소창에 ?search=값 형태로 보입니다. 데이터 크기에 제한이 있고, 검색 등에 사용합니다.

POST 방식

URL 주소 변경 없이 데이터를 숨겨서 전송합니다. 용량 제한이 없고, 로그인/회원가입 등 보안이 필요한 곳에 사용합니다.

input 태그의 type 종류

type 값결과용도
text한 줄 텍스트 입력칸이름, 검색어 등
password비밀번호 입력칸 (●●●)비밀번호
checkbox체크박스 ☑여러 개 선택 가능
radio라디오 버튼 ◉여러 개 중 하나만 선택
file파일 선택 버튼파일 업로드
hidden보이지 않는 입력서버에 보낼 숨김 데이터
button일반 버튼JS 연동용
reset초기화 버튼form 내 입력값 리셋
submit제출 버튼form 데이터 서버 전송
image이미지 형태 제출 버튼커스텀 버튼
email이메일 입력 (형식 검증)이메일 주소
number숫자 입력수량 등
date날짜 선택기생년월일 등
range슬라이더볼륨, 점수 등
color색상 선택기색상 지정

※ email, number, date, range, color는 PPT에 없지만 HTML5에서 추가된 중요 타입입니다.

label 태그 (입력 요소에 이름 붙이기)

<label>의 for 속성과 <input>의 id 속성을 일치시키면, 라벨 클릭 시 해당 입력란에 포커스가 갑니다.

<label for="username">이름</label>
<input id="username" type="text" name="username">

라디오 버튼 (하나만 선택)

같은 그룹의 라디오 버튼은 반드시 name 속성을 동일하게 설정해야 합니다.

<input id="man" type="radio" name="gender" value="m">
<label for="man">남자</label>

<input id="woman" type="radio" name="gender" value="w">
<label for="woman">여자</label>
<!-- name="gender"가 같으므로 둘 중 하나만 선택됨 -->

select (드롭다운 선택)

<!-- 기본 선택 -->
<select>
    <option>김밥</option>
    <option>떡볶이</option>
    <option>순대</option>
</select>

<!-- 다중 선택 (Ctrl+클릭) -->
<select multiple> ... </select>

<!-- 그룹별 분류 -->
<select>
    <optgroup label="HTML5">
        <option>Multimedia Tag</option>
    </optgroup>
    <optgroup label="CSS3">
        <option>Animation</option>
    </optgroup>
</select>

textarea, fieldset

<!-- 여러 줄 텍스트 입력 -->
<textarea cols="50" rows="5">기본 텍스트</textarea>

<!-- 입력 양식 그룹 묶기 -->
<form>
    <fieldset>
        <legend>입력 양식</legend>
        <label for="name">이름</label>
        <input id="name" type="text">
    </fieldset>
</form>

textarea 주의사항

textarea 내부의 공백, 줄바꿈, 탭이 그대로 출력됩니다. 그래서 여는 태그 바로 뒤에 내용을 붙여 써야 합니다.

잘못: <textarea>↵ 내용</textarea> → 앞에 빈 줄과 공백이 보임

올바름: <textarea>내용</textarea>

07

공간 분할 태그 (div, span)

웹 페이지의 영역(범위)을 나누기 위한 태그입니다. 시각적 효과는 없고, CSS를 적용하기 위한 그릇 역할을 합니다.

블록 형식: <div>

한 행 전체를 차지합니다. 끝나면 자동 줄바꿈됩니다. h1~h6, p, ul/ol, table도 블록 요소입니다.

인라인 형식: <span>

내용 크기만큼만 차지합니다. 줄바꿈 없이 옆으로 이어집니다. a, input, b, i 등도 인라인 요소입니다.

<!-- 블록: 각 div가 한 줄씩 차지 -->
<div>첫 번째 블록</div>
<div>두 번째 블록</div>

<!-- 인라인: 옆으로 나란히 표시 -->
<span>첫 번째</span>
<span>두 번째</span>

블록 vs 인라인 태그 구분 (외워두면 좋음)

블록 형식: div, h1~h6, p, ul/ol, table, form

인라인 형식: span, a, input, b, i, img, strong, em

08

시맨틱 태그 (HTML5 구조화)

시맨틱(semantic)이란 "의미가 있는"이라는 뜻입니다. HTML5 이전에는 모든 영역을 <div>로만 나눴기 때문에 코드만 봐서는 어디가 메뉴이고 어디가 본문인지 알 수 없었습니다. 시맨틱 태그를 사용하면 코드에 의미가 부여되어 검색엔진과 스크린 리더가 구조를 이해할 수 있습니다.

태그역할위치
<header>머리말 (페이지 제목, 로고, 소개)페이지 최상단
<nav>내비게이션 (메뉴 링크 모음)header 아래 또는 옆
<section>본문의 장(chapter)이나 절본문 영역
<article>독립적인 콘텐츠 하나section 내부
<aside>본문 흐름과 벗어나는 노트, 사이드바양쪽 사이드
<footer>꼬리말 (저작권, 연락처 정보)페이지 최하단

이전 방식 (div만 사용)

<div><h1>제목</h1></div>
<div><ul>메뉴...</ul></div>
<div>본문...</div>
<div>주소...</div>

HTML5 방식 (시맨틱 태그)

<header><h1>제목</h1></header>
<nav><ul>메뉴...</ul></nav>

시맨틱 태그를 써야 하는 이유

1. 검색엔진(Google)이 페이지 구조를 더 잘 이해하여 검색 순위에 유리합니다 (SEO).

2. 시각장애인용 스크린 리더가 "여기는 내비게이션", "여기는 본문"을 구분할 수 있습니다.

3. 코드 가독성이 좋아져 팀 협업 시 유지보수가 쉬워집니다.

4. 시맨틱 태그도 기능적으로는 <div>와 동일합니다 (블록 요소). 차이는 "의미"뿐입니다.

학습 로드맵: 다음 단계는?

Frontend 학습 순서

1단계 (현재) → HTML5: 웹 페이지의 뼈대(구조)를 만드는 법

2단계 → CSS3: 뼈대에 색상, 크기, 레이아웃 등 디자인 입히기

3단계 → JavaScript: 웹 페이지에 동적인 기능 추가 (클릭, 애니메이션 등)

4단계 → React / Vue: 대규모 웹 애플리케이션 개발 프레임워크

5단계 → Node.js / Spring: 백엔드까지 = Fullstack 개발자!
